整数判断

  1. 余数检查

示例：

function isInteger(x) {
    return typeof x === 'number' && x % 1 === 0;
}
  1. Math库函数

示例：

function isInteger(x) {
    return Math.round(x) === x;
    //return Math.floor(x) === x;
    //return Math.ceil(x) === x;
}
  1. 位操作（无法处理超过32位的数字）

示例：

function isInteger(x) {
    return (x | 0) === x;
}
  1. parseInt()
    不能正确地处理所有的整型数字，parseInt()在解析整数之前强迫将第一个参数解析成字符串。因此使用这种方法将数字转换成整型不是一个好的选择。

示例：

function isInteger(x) {
    return parseInt(x, 10) === x;
}
  1. ECMAScript

示例：

function isInteger(x) {
    return Math.trunc(x) === x;
    //Math.trunc()移除数字的小数部分
}
或：内置函数`Number.isInteger()`

DIV宽度自适应

<div id='left'></div>
<div id='right'></div>
<div id='auto-width'></div>

#left {
    float: left;
    width: 100px;
}
#right {
    float: right;
    width: 100px;
}
#auto-width {
    margin-left: 100px;
    margin-right: 100px;
}
  1. 自适应宽度的层不要指定宽度和float属性
  2. 自适应宽度的层物理的位置放在最下面
  3. 自适应的div中不能出现width: 100%，否则在IE6中会出现错位现象，IE7，IE8，ff3，chrome都正常。
  4. DIV的默认（不写）宽度是100%，但不撑大父框架，写出width: 100%，如果再使用了margin或padding就会将父框架撑大。
  5. Table默认宽度为内容的长度，与DIV不同，因此最好不要混着用，否者这个宽度自适应的方法在IE6下就有问题了。
  6. 如果父框架设定了固定宽度，那么子div的宽度设为width: 100%，即使设定了margin、padding、border都不会撑大父框架。

DOM鼠标事件

  1. mouseover
    鼠标指针经过任何子元素都会触发绑定在父元素上的mouseover事件
  2. mouseout
    鼠标指针离开任何子元素时都会触发父元素上的mouseover事件
  3. mouseenter
    鼠标指针经过绑定的元素时触发事件，经过其子元素时，不会触发事件
  4. mouseleave
    只有当鼠标离开绑定的元素时才会触发该事件，经过其子元素时，不会触发事件
  5. hover != mouseover + mouseout
    hover = mouseenter + mouseleave

元素排序、计数

  1. ol元素的属性reverse

示例：

<ol reversed>
    <li>1</li>
    <li>2</li>
    <li>3</li>
</ol>
  1. counter

示例：

<ol>
    <li>1</li>
    <li>2</li>
    <li>3</li>
</ol>

ol {
    list-style: none;
    counter-reset: increment 4;
    //counter-reset: increment-variable initial-value;
}

ol li:before {
    counter-increment: increment -1;
    //counter-reset: increment-variable increment-step;
    content: counter(increment) ". ";
}
