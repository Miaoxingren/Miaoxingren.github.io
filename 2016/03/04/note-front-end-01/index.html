HTML5全局属性contextmenu

规定元素的上下文菜单：当用户右键点击元素时，会出现上下文菜单。

<element contextmenu='menu_id'></element>

W3C示例：

<div contextmenu="mymenu">
    <menu type="context" id="mymenu">
        <menuitem label="Refresh"></menuitem>
        <menuitem label="Twitter"></menuitem>
    </menu>
</div>

目前只有Firefox支持

常见数据存储方式

IndexdDB

HTML5 的本地存储，把一些数据存储到浏览器（客户端）中，当与网络断开时，可以从浏览器中读取数据，用来做一些离线应用。

Cookie

  1. 通过在客户端(浏览器)记录信息确定用户身份，最大为 4 kb ，IE6大约只能存储2K。
  2. 需要指定作用域，不可以跨域调用。
  3. 每个特定的域名下最多生成20个cookie。
  4. 随着每次HTTP请求头信息一起发送，无形中增加了网络流量。
  5. 安全性问题。如果cookie被人拦截了，就可以取得所有的session信息。
  6. 有些状态不可能保存在客户端。例如，为了防止重复提交表单，我们需要在服务器端保存一个计数器。如果我们把这个计数器保存在客户端，那么它起不到任何作用。

示例：

var cookieHandler = {
    setCookie: function (key, val, exdays) {
        var date = new Date(Date.now() + (exdays * 24 * 60 * 60 * 1000));
        document.cookie = key + "=" + val + ";expires=" + date.toGMTString();
    },
    getCookie: function (key) {
        var cookies = document.cookie.split(';'),
            i = cookies.indexOf(key);
        if (i > 0) {
            return cookies[i].split("=")[-1];
        }
    },
    removeCookie: function (key) {
        setCookie(key, "", -1);
    }
}

URL参数

使用 get 方法，从服务器上获取数据，大小不能大于 2 kb 。

Session Storage

服务器端使用的一种记录客户端状态的机制。IE不支持。
同一个窗口（或者Tab）页面刷新或者跳转，都能获取到本地存储的数据。
当新开窗口或者页面时，原来的数据就失效了。
当用户关闭浏览器窗口后，数据会被删除。

post

向服务器传送数据，数据量较大。

Local Storage

HTML5 的本地存储，将数据保存在客户端中（一般是永久的）。在移动设备上，大部分浏览器都支持web storage特性。Firefox 3.5、Safari 4和IE8支持。

UserData

IE6+浏览器使用userData存储数据，容量可达到640K。这个持久化数据放在缓存中，如果缓存没有清理，那么会一直存在。

Flash ShareObject

能解决cookie存储的两个弊端，能够跨浏览器，需要安装Flash插件，当浏览器没有安装Flash控件时就不能用了。

Google Gear

Google开发出的一种本地存储技术。
缺点：需要安装Gear组件。

Global Storage

类似于IE的userData，Firefox2+支持，IE不支持。
在HTML5中提供了localStorage来取代globalStorage。

使用：

globalStorage[location.hostname]['name'] = 'tugai';
globalStorage[location.hostname]['name'];
globalStorage[location.hostname].removeItem('name');

CSS定位机制

文档流

元素的位置在正常的流中，与position:static位置相同

浮动（float）

元素浮动会脱离原来的文档流。

定位（position）

Static 定位

HTML元素的默认值，即没有定位，元素出现在正常的流中。

Fixed 定位

元素的位置相对于浏览器窗口是固定位置，即使窗口是滚动的它也不会移动，会脱离原来的文档流。

Relative 定位

相对定位元素的定位是相对其正常位置，仍在正常的流中。

Absolute 定位

绝对定位的元素的位置相对于最近的已定位父元素，如果元素没有已定位的父元素，那么它的位置相对于<html>，会脱离原来的文档流。

Object.keys(obj)

Object.keys 返回一个所有元素为字符串的数组，其元素来自于从给定的对象上面可直接枚举的属性。这些属性的顺序与手动遍历该对象属性时的一致。

JavaScript Array filter()方法

array.filter(function(currentValue, index, arr), thisValue)

currentValue 必须，当前元素的值
index 可选，当前元素的索引值
arr 可选，当前元素属于的数组对象
thisValue 可选，对象作为该执行回调时使用，传递给函数，用作 “this” 的值。如果省略了 thisValue ，”this” 的值为 “undefined”。

JavaScript splice()方法

array.splice(index, howmany, item1, ....., itemX)

index 必需，规定从何处添加/删除元素，该参数是开始插入和（或）删除的数组元素的下标，必须是数字。
howmany 必需，规定应该删除多少元素，必须是数字，但可以是 “0”，如果未规定此参数，则删除从 index 开始到原数组结尾的所有元素。
item1, …, itemX 可选，要添加到数组的新元素。

CSS布局

position实现相对定位

父元素：position:relative
子元素：position:absolute

此时子元素相对与父元素定位，而不是相对于<html>

CSS奇偶选择器

奇数选择器：element:nth-child(2n)或element:nth-child(even)

偶数选择器：element:nth-child(2n-1)或element:nth-child(odd)

清除浮动

.clear:after {
    content: " ";
    clear: both;
    display: block;
    font-size: 0;
    visibility: hidden;
    height: 0;
}

宽度自适应

HTML文档中，宽度固定的浮动元素在前，宽度自适应元素在后。

DOM事件阻止取消

DOM中的事件对象：（符合 W3C 标准）

preventDefault() 取消事件默认行为。
stopImmediatePropagation() 取消事件冒泡，会影响当前节点中的事件侦听器。
stopPropagation() 取消事件冒泡，不会影响当前节点 (currentTarget) 中的任何事件侦听器。

IE中的事件对象：

cancelBubble() 取消事件冒泡
returnValue() 取消事件默认行为
IE不支持捕获

获取自定义属性

for in循环可以获取object的所有属性，包括自定义属性以及原型链属性，hasOwnProperty()只能获取自定义属性，无法获取原型链属性。
