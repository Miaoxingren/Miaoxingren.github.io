CSS预处理器

CSS预处理器定义了一种新的语言，其基本思想是，用一种专门的编程语言，为CSS增加了一些编程的特性，将CSS作为目标生成文件，然后开发者就只要使用这种语言进行编码工作。通俗的说，CSS预处理器用一种专门的编程语言，进行Web页面样式设计，然后再编译成正常的CSS文件，以供项目使用。CSS预处理器为CSS增加一些编程的特性，无需考虑浏览器的兼容性问题，例如你可以在CSS中使用变量、简单的逻辑程序、函数等等在编程语言中的一些基本特性，可以让你的CSS更加简洁、适应性更强、可读性更佳，更易于代码的维护等诸多好处。到目前为止，在众多优秀的CSS预处理器语言中就属Sass、LESS和Stylus最优秀。

原型与原型链

JS在创建对象（不论是普通对象还是函数对象）的时候，都有一个叫做proto的内置属性，通常称之为原型，用于指向创建它的函数对象的原型对象prototype。
函数对象的一个属性prototype就是原型对象。Function.prototype除外，它是函数对象，但它很特殊，他没有prototype属性。
原型的值可以是一个对象，也可以是null。
如果它的值是一个对象，则这个对象也一定有自己的原型。这样就形成了一条线性的链，称之为原型链。
访问一个对象的原型可以使用ES5中的Object.getPrototypeOf方法，或者ES6中的proto属性。

运行以下代码：

alert(typeof Object.prototype === 'object');                 //true
alert(typeof Function.prototype === 'function');             //true
alert(typeof Function.prototype.prototype === 'undefined');  //true
alert(Function.prototype.__proto__ === Object.prototype);    //true
alert(Object.__proto__ === Function.prototype);              //true
alert(Function.__proto__ === Function.prototype);            //true
alert(Function.prototype.constructor === Function);          //true
alert(Object.prototype.constructor === Object);              //true
alert(Function.constructor === Function);                    //true
alert(Object.constructor === Object);                        //true

var Person = function (name) {
    this.name = name;
}
Person.prototype.getName = function () {
    return this.name;
}
var anny = new Person("Anny");

alert(anny.__proto__ === Person.prototype);                   //true
alert(typeof Person.prototype === 'object');                  //true
alert(Person.__proto__ === Function.prototype);               //true
alert(Person.constructor === Person);                         //true
alert(Person.prototype.constructor === Person);               //true
alert(Person.prototype.__proto__ === Object.prototype);       //true
alert(Object.prototype.__proto__ === null);                   //true

函数表达式和函数声明

函数表达式function functionName() {}
函数声明var functionName = function(){}
函数的声明的解析是在预执行(pre-execution)阶段，也就是浏览器准备执行代码的时候。因此，通过函数声明来定义函数，可以在定义前或后被调用。
函数表达式会在执行流(execution flow)执行到它的时候才会创建函数。所以，函数表达式必须被执行了（此时函数才定义了）才能被调用。

Javascript闭包

Javascript里的function能访问它们的：

  1. 参数
  2. 局部变量或函数
  3. 外部变量，包括
    3.1 全局变量，包括DOM
    3.2 外部函数的变量或函数。

如果一个函数访问了它的外部变量，那么它就是一个闭包。
注意，外部函数不是必需的。通过访问外部变量，一个闭包可以维持（keep alive）这些变量。在内部函数和外部函数的例子中，外部函数可以创建局部变量，并且最终退出；但是，如果任何一个或多个内部函数在它退出后却没有退出，那么内部函数就维持了外部函数的局部数据。
闭包可以用于创建含有隐藏数据的函数。
闭包是一种设计原则，它通过分析上下文，来简化用户的调用，让用户在不知晓的情况下，达到他的目的。

property和attribute

property是DOM中的属性，是JavaScript里的对象中的一个属性；
attribute是HTML标签上的特性，它的值只能够是字符串。
在IE6\7\8里面，propertites与attributies是自动同步的，并且attributies的值不限于字符串。
在IE6\7里面，propertites与attributies的键名不区分大小写。

attribute

attribute是一个特性节点，每个DOM元素都有一个对应的attributes属性来存放所有的attribute节点，attributes是一个类数组的容器，说得准确点就是NameNodeMap，总之就是一个类似数组但又和数组不太一样的容器。
Attribute会显示在DOM上面，它的键名不区分大小写。
与Property不同，Attribute会DOM节点上显示出来，但不会在DOM对象中被for-in遍历出来。
attributes的每个数字索引以名值对(name="value")的形式存放了一个attribute节点。
elem.attributes[index].name
elem.attributes[index].value
但是IE6-7将很多东西都存放在attributes中，上面的访问方法和标准浏览器的返回结果又不同。
通常要获取一个attribute节点直接用getAttribute方法：
elem.getAttribute(attributeName)
要设置一个attribute节点使用setAttribute方法，要删除就用removeAttribute：
elem.setAttribute(attributeName, attributeValue)
elem.removeAttribute(attributeName)
attributes是会随着添加或删除attribute节点动态更新的。

property

property就是一个属性，如果把DOM元素看成是一个普通的Object对象，那么property就是一个以名值对(name="value")的形式存放在Object中的属性。
每一个DOM节点，都是一个对象。像其他JS对象一样，DOM节点这类型HTMLElement对象，也可以添加一些方法或者属性。这些自定义添加的属性，就是property。（它能被JS的for-in方法遍历出来，但innerHTML里面不会显示）
要添加和删除property也简单多了，和普通的对象没啥分别：
elem.propertyName = propertyValue; // 添加
console.log( elem.propertyName ) // 获取
delete elem.propertyName // 删除

DOM元素一些默认常见的attribute节点都有与之对应的property属性，比如div元素的id和class既是attribute，也有对应的property，不管使用哪种方法都可以访问和修改。比较特殊的是一些值为Boolean类型的property，如一些表单元素，对于这些特殊的attribute节点，只有存在该节点，对应的property的值就为true：

<input type="radio" checked="checked" id="raido">
var radio = document.getElementById( 'radio' );
console.log( radio.getAttribute('checked') ); // checked
console.log( radio.checked ); // true

但是对于自定义的attribute节点，或者自定义property，两者就没有关系了。
对于IE6-7来说，没有区分attribute和property。
所有的DOM节点对象，都有一套标准的properties。这些DOM对象的标准properties，会与其attributes自动同步，但值并不完全相等。
只能从Attribute单向同步到Property，Property却不能同步到Attribute。

从输入URL到页面呈现

  1. 浏览器向Web缓存器请求，如果Web缓存器存有请求的对象则发送响应，如果没有则向Web服务器请求
  2. 浏览器查找域名的IP地址（浏览器缓存->系统缓存->路由器缓存->ISP DNS 缓存->递归搜索）
  3. 浏览器向 web 服务器发送一个 HTTP 请求
  4. 服务器的永久重定向响应（从 http://example.com 到 http://www.example.com）
  5. 浏览器跟踪重定向地址,发送另一个获取请求
  6. 服务器处理请求，返回一个 HTTP 响应
  7. 浏览器解析HTML，渲染呈现页面
  8. 浏览器发送请求获取嵌入在 HTML 中的资源（如图片、音频、视频、CSS、JS等等）
